03Matching Engine — How Matches Are Made
🔍Match Request Triggered
User opens "Discover" → Next.js calls GET /matches/recommended → Express validates JWT → Query builder assembles filters from partner_preferences table.
↓
🗄️PostgreSQL Candidate Pool
GIN index on text arrays (caste[], religion[]), BTREE on age/height. Filters: opposite gender, active account, non-blocked users, within preference bounds. Returns up to 200 candidates.
↓
🧮Score Engine (Node.js Service)
Each candidate scored 0–100. Scores cached in Redis (TTL 30 min). Sorted ranked list returned. Premium verified profiles get display boost (not score manipulation).
↓
🎂
Age Compatibility
Requested age range vs actual DOB. Linear decay penalty for outliers.
weight: 20 pts
📍
Location Proximity
City → State → Country hierarchy. Same city = full score.
weight: 15 pts
🛐
Religion & Caste
Exact match = full. open_to_any_caste bypasses caste filter.
weight: 20 pts
🎓
Education Level
Enum-ranked comparison. Close rank gap = higher score.
weight: 15 pts
💰
Income Match
Checks annual income meets partner's minimum threshold.
weight: 10 pts
👨‍👩‍👧
Family Values
Traditional / Moderate / Liberal alignment scoring.
weight: 10 pts
📏
Height Match
Validated against height_min / height_max preference range.
weight: 5 pts
✅
Verified Boost
Verified identity +3 pts. Active last 7 days adds +2 pts.
weight: 5 pts
04Interest → Connect → Rishta Flow
💌User A Sends Interest
POST /interests/send → creates record (status=PENDING). Socket.io event interest:received emitted to User B if online. SMS/email fallback if offline.
↓
Does User B accept the Interest?
✓ ACCEPTED
status → ACCEPTED. Chat thread auto-created. Both users notified. Thread unlocked immediately.
✗ DECLINED
status → DECLINED or expires after 30 days. User A notified. Profile still visible but chat stays locked.
↓
💬Chat Thread Opens (Socket.io Room)
Socket.io room thread:{uuid} created. Both users auto-joined. Messages persisted to PostgreSQL. Phone numbers remain hidden until mutual consent.
↓
📞Contact Exchange & Meeting
Both users must tap "Share Contact" → mutual consent flag stored in DB → phone numbers unlocked. Video call option available (Diamond plan). Families can join via shared profile link.
↓
💍Rishta Fixed — Success!
Users mark "Match Found". Profile deactivated. Celebratory email sent. Success story submitted to admin queue for publishing on homepage marketing page.

⚡ Chat Features
⚡
Real-Time Messaging
Socket.io WebSocket rooms per match thread. Sub-100ms message delivery. Auto-reconnect on drop.
✓✓
Read Receipts
Single tick (sent) → double tick (delivered) → blue tick (read). Stored in messages.is_read + read_at.
✍️
Typing Indicator
Socket event user:typing broadcast to thread room. Auto-clears after 3s of inactivity.
🖼️
Photo & File Sharing
Upload to Cloudinary (max 10MB). msg_type = IMAGE | FILE | AUDIO. Thumbnail previews inline.
🔒
Contact Lock / Unlock
Phone numbers hidden until BOTH users tap "Share Contact". Mutual consent flag persisted to DB.
🗑️
Delete & Block
Delete for self or both parties. Block closes thread and hides profile. Reports go to admin queue.
📹
Video Call (Gold+)
WebRTC peer-to-peer video call from chat header. Gold & Diamond plans only. No recording stored.
🔕
Mute, Archive & Star
Mute per thread, archive old chats, star important messages. Unread badge synced via Redis.
📤 Client → Server Events
chat:join
Join thread room on open. Marks all messages delivered.
chat:send_message
Sends {threadId, content, msg_type}. Server saves to PG & broadcasts.
chat:typing
Fired on keypress. Debounced 500ms. Clears on blur or send.
chat:read
Marks messages read. Server updates DB & emits receipt to sender.
chat:delete_message
Soft delete: is_deleted=true. Receiver sees "Message deleted".
📥 Server → Client Events
chat:new_message
Broadcast to thread room. Updates UI live. Push notif if user offline.
chat:message_read
Sent to message sender. Turns double tick blue in UI.
interest:received
Notifies user B of new interest. Shows banner in dashboard.
user:online_status
Broadcasts presence. Updates "Online now" / "Last seen X ago".
chat:typing_indicator
Shows "X is typing..." bubble. Clears after 3s timeout.
